## 1. 产品愿景

### 1.1 终极愿景

将 AI 从「对话式代码生成工具」升级为**集效率工具、全维度可视化、结构化系统空间、可视化认知界面、情绪化交互伙伴、可演化数字生态于一体的下一代软件构建范式**。

**把所有可视化的部分变得可视化**——囊括开发者、创业者、生产者从**模糊 idea → 成熟产品**的完整路径，让 vibe coding 过程 **as easy as possible**。

### 1.2 全路径产品地图

```
Idea 模糊构想
    ↓
商业分析（用户画像 / 场景 / 细分市场）        ← Phase 2
    ↓
产品对齐（生图/生视频 → 最终产品预览）          ← Phase 2
    ↓
架构规划（功能地图骨架 / 架构对话）             ← MVP
    ↓
协同开发（Workspace / 任务认领 / 共同编辑）     ← Phase 2
    ↓
可视化构建（三大可视化 + Scratch 逻辑块）       ← MVP → Phase 3
    ↓
AI 协作（Agent 流程可见 / Harness 根因修复）   ← MVP
    ↓
测试 / 部署 / 迭代                            ← Phase 2+
    ↓
成熟产品
```

### 1.3 演进路径总览

```
Phase 1 (MVP)              Phase 2 (Growth)           Phase 3 (Scale)          Phase 4 (Vision)
转码小 vibe coder      →   全部 vibe coder       →   团队 + 生态萌芽      →   AR + Agent 网络
三大可视化 + Harness      商业分析 + 对齐视频         Scratch 逻辑 + RAG        能力市场 + JARVIS
                         + Workspace 协同
```

### 1.4 一句话定位（随阶段演进）

| 阶段 | 定位 |
|------|------|
| MVP | 三大可视化 + Harness 的 Cursor 式 AI 开发环境 |
| Growth | 从 idea 到产品的全路径 Project OS |
| Scale | 团队协作的可视化软件构建平台 |
| Vision | vibe coding 默认范式 + AI 协作能力生态 |

### 1.5 产品交付形态（全阶段不变）

- **独立桌面应用**（Electron / Tauri），内置 Monaco 编辑器
- **不做 IDE 插件**，UI/UX 对标 Cursor
- 主编辑区可切换多种可视化视图 + 代码视图

## 2. 问题陈述（全生命周期）

### 2.1 行业共性痛点

| 问题 | 表现 | 影响 | 首次解决阶段 |
|------|------|------|-------------|
| 上下文迷失 | 长对话后 AI 不知道在改什么 | 改错文件、重复劳动 | MVP |
| 修改定位难 | 用户说不清「改哪个功能」 | prompt 越来越长 | MVP |
| 架构恐惧 | 转码用户看不懂技术架构 | 不敢改、全靠 AI 瞎改 | MVP |
| 需求表达模糊 | 自然语言歧义大 | AI 理解偏差、返工 | MVP |
| 报错处理混乱 | debug 与业务需求混聊 | 效率低、挫败感强 | MVP |
| Agent 黑盒 | 不知道 AI 正在做什么 | 无法纠错，反复说「看一下哪里有问题」 | MVP |
| 打补丁式修 Bug | AI 覆盖报错而非找根因 | 问题复发、技术债累积 | MVP |
| File Tree 迷失 | 文件夹逐层展开，只见文件不见结构 | 找不到问题文件 | MVP |
| 商业构想模糊 | 创业者不知为谁做、做什么 | 方向漂移、无效开发 | Phase 2 |
| 产品形态不可见 | 不知最终产品长什么样 | 做出来与预期不符 | Phase 2 |
| 协作困难 | 多人改同一项目混乱 | 冲突、重复劳动 | Phase 2 |
| 架构失控 | 项目变大后结构混乱 | 资深用户也迷失 | Phase 2 |
| 业务逻辑不可见 | 代码里藏了看不懂的逻辑 | 非技术用户无法参与 | Phase 3 |

### 2.2 产品解法总览

| 痛点类别 | 解法 |
|----------|------|
| 定位 / 理解 | 三大可视化 + 点 node context 注入 |
| 需求模糊 | Prompt 精进反馈 |
| Agent 不可控 | Agent 流程可视化 + 步骤纠错 |
| 修 Bug 低效 | Debug Harness 根因修复 + debug/business 分离 |
| 学习成本高 | 新手指导 + Cursor 式 familiar 布局 |
| 商业 / 产品模糊 | 商业模型可视化 + 产品对齐预览（Phase 2） |
| 协作 | Workspace + 任务认领（Phase 2） |
| 业务逻辑 | Scratch 逻辑块（Phase 3） |

## 3. 核心战略：全可视化

### 3.1 可视化支柱

| 可视化层 | 看见什么 | 解决什么 | 阶段 |
|----------|----------|----------|------|
| **软件结构可视化** | 功能关系、模块关系 | 不知道改哪个功能 | MVP P0 |
| **Agent 流程可视化** | AI 正在做什么、每步输入输出 | Agent 黑盒、无法纠错 | MVP P0 |
| **File Tree 可视化** | 语义树状文件结构全貌 | 文件夹逐层展开迷失 | MVP P0 |
| **商业模型可视化** | 用户画像、场景、细分市场 | 创业者不知为谁做 | Phase 2 |
| **产品对齐可视化** | 生图/生视频的产品预览 | 不知最终产品长什么样 | Phase 2 |
| **Scratch 逻辑可视化** | 业务逻辑块 | 业务逻辑不可见 | Phase 3 |
| **3D 云图 / AR** | 大规模项目空间导航 | 节点过多 2D 混乱 | Phase 3–4 |

### 3.2 首次商业化必推：三大可视化

以下三项构成**首次付费上线**的核心交付（详见 MVP-PRD §4.3）：

| 可视化层 | 用户价值 |
|----------|----------|
| 软件结构可视化 | 看见功能关系，点选修改 |
| Agent 流程可视化 | 看见 AI 在干什么，随时纠错 |
| File Tree 可视化 | 看见项目文件结构全貌，快速定位 |

可由**一个 Agent 统管**或**三个专项 Agent 分工**，产品层面对用户呈现为统一体验。

### 3.3 三大可视化联动原则

任一可视化层中的点选操作，均触发：

1. 右侧 AI Chat **context 自动切换**（200ms 内）
2. 另外两个可视化层**同步高亮**关联节点
3. Agent 流程面板**标注当前操作来源**

### 3.4 Agent 架构策略

| 阶段 | 策略 | 说明 |
|------|------|------|
| MVP | 单 Agent + 流程外显 | 一个 Agent 统管，流程图是其内部 step 的外显 |
| Phase 2 | 可选三 Agent 分工 | 结构 Agent / 执行 Agent / Debug Agent，流程图展示协作 |
| Phase 3+ | Multi-Agent 编排 | 对用户仍呈现统一陪伴体 |

## 5. 用户旅程（全生命周期）

### 5.1 Phase 1 旅程（MVP）

**路径 A — 已有项目**：

```
打开应用 → 导入 repo → 功能地图 + File Tree 语义树生成
→ 点击功能节点 → context 聊天 → Prompt 精进 → 确认
→ Agent 流程面板实时展示 → AI 执行 → 代码 review
→ 报错时 Harness 根因修复（非打补丁）
```

**路径 B — 从零开始**：

```
新建项目 → 架构对话 → 功能地图骨架 → 逐节点开发（同路径 A）
```

**Agent 纠错旅程**（MVP 核心差异化）：

```
修 Bug 触发 → Agent 流程展示：诊断 → 假设根因 → 修复 → 验证
→ 用户发现 Agent 定位文件有误 → 点击流程步骤注入纠正
→ Harness 拒绝补丁式修复 → 根因修复后三大可视化同步变绿
```

> 完整用户故事见 [MVP-PRD.md](./MVP-PRD.md) §5.3–5.4

### 5.2 Phase 2 旅程（Growth）

```
创建 Workspace → 从商业分析开始（Persona / 场景 / 细分市场）
→ 生成产品对齐预览（UI 图 + 演示视频）
→ AI 拆分功能地图 → 成员认领节点
→ 各自节点 context 开发（Agent 流程团队可见）
→ 对齐预览 vs 实际实现对比 → 迭代
```

### 5.3 Phase 3+ 旅程

- 功能节点 drill-down 到 Scratch 逻辑块编辑
- 架构视图下拖拽重构 + 双真相层
- 团队实时协作 + 决策日志

### Phase 2：Growth（12–24 个月）

#### 6.6 阶段目标

- 用户群拓展至创业者、indie hacker、非技术产品方
- 实现 Idea → 产品路径的**上游段**（商业分析 + 产品对齐）

- 10,000+ 注册用户，1,000+ 付费用户

#### 6.7 商业模型可视化（新增 ★）

**功能定义**：在开发之前，帮助创业者/产品方将商业构想结构化、可视化。

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 商业画布视图 | 用户画像、痛点、价值主张、渠道、收入模型 | P0 |
| 用户画像卡片 | 自动生成 + 用户编辑 Persona | P0 |
| 应用场景列表 | 具体 use case，链接到功能地图节点 | P0 |
| 细分市场分析 | AI 辅助 TAM/SAM/SOM 与竞品定位 | P1 |
| 商业 → 功能映射 | 每个应用场景对应功能节点，双向链接 | P0 |

**交互**：新建项目可选「从商业分析开始」→ 填/聊商业画布 → 自动生成功能地图骨架。

#### 6.8 产品对齐视频/图像生成（新增 ★）

**功能定义**：接入生图/生视频模型，生**最终产品对齐预览**——让 vibe coder 知道未来产品具体要做成什么样。

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 产品预览图生成 | 基于功能描述生成 UI mockup | P0 |
| 产品对齐视频生成 | 基于用户旅程生成 15–30s 演示视频 | P0 |
| 对齐预览 ↔ 功能地图联动 | 预览界面区域对应功能节点 | P0 |
| 版本对比 | 「当前实现」vs「对齐目标」并排 | P1 |
| 导出分享 | 预览图/视频导出，用于融资/团队对齐 | P1 |

#### 6.9 Workspace 协同编辑（新增 ★）

**功能定义**：参考 Notion，创建 Workspace，成员共同编辑开发。项目架构可拆分，成员**认领开发任务**。

| 功能 | 描述 | 优先级 |
|------|------|--------|
| Workspace 创建 | 组织 / 团队 / workspace 三级 | P0 |
| 成员邀请与权限 | Owner / Editor / Viewer | P0 |
| 功能节点任务认领 | 功能地图上认领节点，显示负责人 | P0 |
| 任务看板 | 待办 / 进行中 / 完成 / 报错 | P0 |
| 实时协同 | 功能地图、商业画布 CRDT 多人编辑 | P1 |
| Agent 流程共享 | 团队成员查看彼此 Agent 流程 | P1 |
| 决策日志协作 | 多人决策记录、评论、@mention | P1 |
| per-user 陪伴体 | 独立 AI 风格 + 共享项目结构 | P0 |

**协同流程**：

```
创建 Workspace → 导入/新建项目 → AI 拆分功能地图
    ↓
创始人填写商业画布 + 生成产品对齐预览
    ↓
成员认领功能节点 → 各自节点 context 开发
    ↓
Agent 流程 / Harness 修复对所有成员可见
    ↓
功能节点完成 → 看板更新 → 对齐预览 vs 实际实现对比
```

## 7. 核心功能规格（全生命周期）

### 7.1 软件结构可视化（MVP 起）

### 7.2 Agent 流程可视化（MVP 起）

| 阶段 | 能力 |

**Harness 流程**（全阶段核心）：

## 13. 核心设计原则（全生命周期不变）

1. **能可视化的一定可视化**——软件结构、Agent 行为、文件组织、商业逻辑、产品形态
2. **代码是真相**——所有可视化是投影层，不颠倒
3. **功能优先于架构**——新手看功能关系，资深用户随阶段 unlock 架构深度
4. **点哪改哪**——任一可视化层点选即可注入 context
5. **Agent 过程透明**——用户必须能看见 AI 在做什么，并能纠错；永远不需要说「看一下哪里有问题」
6. **根因修复，拒绝补丁**——Harness 保证 AI 理解问题根源，不覆盖报错
7. **确认理解再执行**——Prompt 精进是全生命周期核心交互
8. **Cursor 心智不丢**——布局 familiar，学习成本最小化
9. **渐进式复杂度**——产品随用户经验增长而展开
10. **Idea 到产品一条链**——从商业分析到部署，最终全部在同一 Workspace 完成
11. **独立产品**——不做 IDE 插件
12. **vibe coding as easy as possible**——新手指导 + 可视化降低一切门槛
